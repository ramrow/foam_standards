# Error Diagnosis and Resolution

## Error Analysis

Looking at the current error log, the simulation is **actively running** — the solver is computing through PIMPLE iterations (currently at iteration 10 within a single time step), residuals are very small (Ux ~3e-05, Uz ~4.5e-06, h ~5.5e-06), and continuity errors are tiny (~1e-09). There are no crashes, no negative values, and no missing keywords.

The sole issue is that the solver **"did not complete (no 'End' marker found)**." This means the simulation ran out of time — it never reached the `endTime` of 1000 seconds because it couldn't finish all 1000 time steps within the wall-clock time limit.

Key observations:
- At iteration 10, residuals are already very good (comparable to iteration 50 in Attempt 4)
- The solver is doing **many PIMPLE iterations per time step** (reaching iteration 10+) when it could likely converge in 2-3
- With `nOuterCorrectors = 10` and up to 10 iterations used, only ~1 time step worth of computation is shown before the log cutoff

The fundamental problem is that **each time step takes too long** due to excessive PIMPLE iterations. The residuals are already very small at iteration 2-3, but the solver continues iterating unnecessarily.

## Proposed Solutions

### 1. Reduce `nOuterCorrectors` significantly

Change `nOuterCorrectors` from `10` to **3** in `system/fvSolution`. Since residuals are already very small at the first few iterations, 3 outer correctors should be sufficient for convergence while keeping each time step computationally affordable:

```
PIMPLE
{
    momentumPredictor   yes;
    nOuterCorrectors    3;
    nCorrectors         2;
    nNonOrthogonalCorrectors 0;

    pRefCell            0;
    pRefValue           0;

    residualControl
    {
        U               1e-05;
        p_rgh           1e-05;
        h               1e-05;
        k               1e-05;
        epsilon         1e-05;
        rho             1e-05;
    }
}
```

### 2. Relax the `residualControl` tolerances to allow faster convergence

Increase the tolerances so the solver can exit the PIMPLE loop sooner. Change from `1e-05` to **`1e-04`**:

```
residualControl
{
    U               1e-04;
    p_rgh           1e-04;
    h               1e-04;
    k               1e-04;
    epsilon         1e-04;
    rho             1e-04;
}
```

### 3. Increase `nCorrectors` slightly to maintain accuracy

Keep `nCorrectors 2` (good for pressure-velocity coupling), and increase `nNonOrthogonalCorrectors` from `0` to **1** to handle the non-orthogonal mesh geometry properly:

```
nCorrectors         2;
nNonOrthogonalCorrectors 1;
```

### Rationale

The simulation logic is sound — it's just too slow per time step. By reducing `nOuterCorrectors` to 3 and relaxing the tolerances to `1e-04`, each time step should complete quickly (typically 2-3 outer iterations instead of 8-10). This will allow the solver to advance through all 1000 time steps and reach the `endTime` of 1000 seconds, at which point it will naturally terminate with the "End" marker.

The key insight from the history is that the solver converges rapidly initially (iteration 1 residuals are already ~1e-05 for U, ~1e-06 for h). The current settings force it to waste iterations trying to reach machine-level precision, which is unnecessary for this transient buoyant flow simulation.
