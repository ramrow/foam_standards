## Error Analysis

The simulation is crashing due to the CrossPowerLaw viscosity model producing extreme values (leading to Courant numbers of ~10^75 and floating point exceptions). The root cause is that the CrossPowerLaw model with `nuInf=10` produces extremely high viscosities, causing numerical instability.

Let me examine the current `momentumTransport` file more carefully. It currently has:

```
laminar
{
    laminarModel    generalisedNewtonian;

    viscosityModel  CrossPowerLaw;

    CrossPowerLawCoeffs
    {
        nuInf   10;
        m       0.4;
        n       3;
    }
}
```

The issue is that `nuInf=10` is far too large for this flow. For a CrossPowerLaw model, `nuInf` represents the infinite-shear-rate viscosity. With a characteristic velocity of 1.5 m/s over a length scale of about 4 meters (from x=-1 to x=3), the shear rate is moderate, and `nuInf=10` creates viscosities orders of magnitude larger than the base `nu=1e-03`, leading to massive resistance and the simulation blowing up.

## Proposed Fix

### Reduce `nuInf` in `constant/momentumTransport`

Change `nuInf` from `10` to a much smaller value compatible with the base kinematic viscosity of `1e-03`. A reasonable starting point would be `nuInf = 0.001` (the same order of magnitude as the base viscosity):

```
CrossPowerLawCoeffs
{
    nuInf   0.001;
    m       0.4;
    n       3;
}
```

If that's not sufficient, progressively reduce it further (e.g., `0.0001`, `0.00001`) until the simulation runs stably. The key insight is that `nuInf` should be on the same order as the base kinematic viscosity (`1e-03`) since the CrossPowerLaw model scales the effective viscosity relative to this baseline.

### Alternative approach if reducing `nuInf` alone doesn't help

Since the user requirement states "use constant viscosity model with nu value of 1e-03", another stable approach would be to **remove the CrossPowerLaw model entirely** and just use the constant viscosity model.
