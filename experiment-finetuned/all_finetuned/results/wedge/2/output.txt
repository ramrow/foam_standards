## Root Cause

The issue is the **vertex ordering**. In OpenFOAM's `hex` block, the first four vertices define one face and the second four define the opposite face. The normal of the first face must point **toward** the second face. With the current vertices:

- Face 0,1,2,3: all at z=-0.005
- Face 4,5,6,7: all at z=0.005

So the second face is "above" the first. The normal of face (0→1→2→3) using the right-hand rule should point in the **+z direction** (toward the second face). But looking at the coordinates:
- 0: (-0.15242, -0.005, -0.005)
- 1: (-0.15242, 0.1524, -0.005)
- 2: (0, 0.1524, -0.005)
- 3: (0, -0.005, -0.005)

Edge 0→1 = (0, 0.1574, 0), Edge 0→3 = (0.15242, 0, 0). Cross product (0→1)×(0→3) = (0·0 - 0·0, 0·0.15242 - 0·0, 0·0 - 0.1574·0.15242) = (0, 0, -0.0240). The normal points in **-z**, which is away from the second face (at +z). This is why the block has inward-pointing faces.

The vertex ordering needs to be **reversed** so that the normal points in the correct direction. The correct ordering should be:
