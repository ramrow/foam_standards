## Root Cause Analysis

The issue is that the **initial and boundary conditions produce extreme density gradients near the step**, especially where the supersonic inflow (M≈3) hits the backward-facing step corner. This creates localized regions where pressure drops sharply, potentially going negative.

The key problem is the **waveTransmissive boundary condition on the outlet** combined with the **supersonic inflow**. In `rhoCentralFoam`, the outlet boundary condition needs careful handling because the characteristic speeds (including the speed of sound) determine whether waves propagate out of the domain. At Mach 3, the flow is highly compressible and sensitive.

Additionally, looking at the reference forwardStep tutorial more carefully, the outlet pressure BC should be compatible with the supersonic outflow condition. The `waveTransmissive` BC uses `gamma 1.4`, but the thermodynamic properties define `specie` without explicitly setting `gamma` in the thermo model — there could be a mismatch.

## Proposed Solutions

### Solution 1: Change the outlet pressure boundary condition

Replace the `waveTransmissive` BC on the `outlet` in `0/p` with a **`totalPressure`** boundary condition, which is more robust for supersonic outflow:

```
outlet
{
    type            totalPressure;
    p0              uniform 1;
    value           uniform 1;
}
```

This avoids potential issues with `field`, `psi`, and `lInf` parameters that may conflict with the `hePsiThermo` setup, and `totalPressure` is the standard BC used in the forwardStep tutorial for the outlet when dealing with supersonic inflow.

### Solution 2: Add a relaxation factor in `fvSolution`

Add under-rexpression to stabilize the solution. Modify the `fvSolution` to include relaxation for `rho`, `rhoU`, and `rhoE`:

```
solvers
{
    "(rho|rhoU|rhoE)"
    {
        solver          diagonal;
        relTol          0.1;
        tolerance       1e-05;
        nRelax          0.7;
    }
}
```

This adds 30% under-relaxation to prevent overshooting in the strongly compressible region near the step.

### Recommended Priority

**First apply Solution 1** (change outlet BC to `totalPressure`), as this is the most likely source of the instability. The `waveTransmissive` BC can produce non-physical reflections or incorrect wave propagation at the outlet for this high-Mach-number case, leading to negative pressures. If that doesn't resolve it, add the relaxation from Solution 2.
