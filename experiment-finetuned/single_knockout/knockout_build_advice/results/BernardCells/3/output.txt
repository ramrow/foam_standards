Looking at the error message more carefully:

```

    while reading object p_rgh
```

This means the solver is reading `p_rgh` and expecting it to be a `volScalarField`, but the file declares it as `volVectorField`. This is the opposite of what we want — the solver wants `p_rgh` to be a scalar field, but the hydrostatic initialisation needs it to be a vector to store `rho*g*h`.

The solution is to make `p_rgh` a `volScalarField` (so the solver reads it correctly), but ensure that the hydrostatic initialisation can still work. The hydrostatic initialisation computes `p_rgh = p - rho*g*h`. If `rho*g*h` is a vector, we need `p_rgh` to be a vector to store the result. But the solver reads `p_rgh` as a scalar.

The real fix is to ensure that the hydrostatic initialisation computes `rho*g*h` as a *scalar* field, not a vector. This means the height field `h` must be treated as a scalar, not a vector.
