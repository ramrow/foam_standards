# Error Diagnosis

## Error Analysis

The error is a **floating point exception (SIGFPE)** occurring in the `kEpsilon` turbulence model's `correctNut()` function:

```
#3  Foam::divide(Foam::Field<double>&, Foam::UList<double> const&, Foam::UList<double> const&)
#5  Foam::RASModels::kEpsilon<Foam::incompressibleMomentumTransportModel>::correctNut()
```

This happens when dividing by zero — specifically, the `kEpsilon` model is trying to compute turbulent viscosity (`nuEff`) but encountering a division by zero, which triggers the SIGFPE signal.

## Root Cause

The simulation is diverging. The `kEpsilon` model produces wildly oscillating values that grow unbounded, eventually causing numerical overflow leading to the floating point exception. This is a classic instability problem in lid-driven cavity simulations.

Looking at the current setup:
- The lid moves at velocity `(1 0 0)` — this creates strong shear-driven turbulence.
- With `deltaT = 0.005` and only 2 correctors, the PISO algorithm may not be stable enough for this aggressive transient.
- The initial transient from rest to the moving lid generates very high velocities, which combined with the k-Epsilon model can cause rapid growth of turbulent kinetic energy and dissipation rate.

## Proposed Solutions

### 1. Increase PISO Correctors for Stability

Change `nCorrectors` from `2` to `3` or even `4` in the `fvSolution`:

```
PISO
{
    nCorrectors     4;
    nNonOrthogonalCorrectors 0;
    ...
}
```

More correctors mean the pressure equation is solved more accurately per time step, improving stability.

### 2. Reduce Time Step Size

Reduce `deltaT` significantly to improve stability. Try `deltaT 0.001` or even `0.0005`:

```
deltaT          0.001;
```

With fewer cells (20×20) and a small domain, a much smaller time step is often necessary for explicit coupling between the velocity field and the k-Epsilon turbulence model.

### 3. Initialize Fields Properly

Set non-zero initial values for `k` and `epsilon` to avoid extremely large transients at startup:

- In `0/k`: Set `internalField` to a reasonable value like `uniform (0.1 0.1 0.1);` or derive from the lid velocity.
- In `0/epsilon`: Similarly initialize to a reasonable value.

A common initialization for the k-Epsilon model in a cavity is:
- `k` ~ 0.01–0.1 (depending on velocity scale)
- `epsilon` ~ 0.001–0.1

### 4. Relaxation Factors

Add relaxation factors in `fvSolution` to stabilize the solution:

```
solutions
{
    p
    {
        solver          PCG;
        preconditioner  DIC;
        tolerance       1e-06;
        relTol          0.01;
        nRelax          0.7;
    }
    U
    {
        solver          PBiCGStab;
        preconditioner  DILU;
        tolerance       1e-05;
        relTol          0.1;
        nRelax          0.7;
    }
    k
    {
        solver          PBiCGStab;
        preconditioner  DILU;
        tolerance       1e-05;
        relTol          0.1;
        nRelax          0.7;
    }
    epsilon
    {
        solver          PBiCGStab;
        preconditioner  DILU;
        tolerance       1e-05;
        relTol          0.1;
        nRelax          0.7;
    }
}
```

### Recommended Priority Order

1. **First**, reduce `deltaT` to `0.001` — this is the most critical fix.
2. **Second**, increase `nCorrectors` to `4`.
3. **Third**, add proper initialization for `k` and `epsilon` (non-zero values).
4. **Fourth**, add relaxation factors as above.

The combination of a smaller time step and more PISO iterations should resolve the divergence that leads to the SIGFPE.
