# Error Diagnosis and Resolution

## Error Analysis

The simulation is diverging catastrophically. Here's what the error log shows:

1. **`k` equation**: The initial residual is 1.0 and the final residual explodes to `9.40246e+143` after 1000 iterations (the maximum allowed), indicating the turbulent kinetic energy is blowing up.

2. **`p` equation**: The GAMG solver also fails with a final residual of `7.26274e+12`, showing pressure coupling is completely unstable.

3. **Courant number**: At Time = 0.0002s, the Courant number has already reached **7.48915**, far exceeding 1. This means the time step is too large for the current flow conditions, causing numerical instability.

4. **Floating point exception (sigFpe)**: The solver crashes due to division by zero or overflow from the unbounded fields.

## Root Causes

### 1. Incorrect Turbulence Model Parameters (`k` and `epsilon`)
The `k` and `epsilon` values in the `0/k` and `0/epsilon` files are way too high for this small channel flow:
- `k` is set to **0.06** — this is appropriate for ~1 m/s flows in larger domains, but here it creates excessive turbulence energy.
- `epsilon` is set to **14855** — this is astronomically high and causes immediate divergence.

For a 15 m/s inlet velocity in a small channel, the turbulence intensity should be much lower. Looking at the pitzDaily tutorial, typical values are around `k = 0.00375` and `epsilon = 0.09`.

### 2. Time Step Too Large
Even though `deltaT = 0.0001` is specified, the simulation still develops enormous Courant numbers because the initial transient blows up. However, the fundamental issue is the incorrect turbulence initialization.

## Recommended Fixes

### Fix 1: Correct `k` and `epsilon` Values in Initial Condition Files

In **`0/k`**:
- Change `internalField` from `uniform 0.06` to `uniform 0.00375`
- Change inlet `fixedValue` from `uniform 0.06` to `uniform 0.00375`

In **`0/epsilon`**:
- Change `internalField` from `uniform 14855` to `uniform 0.09`
- Change inlet `fixedValue` from `uniform 14855` to `uniform 0.09`
- Change wall function values (`epsilonWallFunction`) from `uniform 14855` to `uniform 0.09`

These values come from scaling the standard pitzDaily tutorial appropriately for the given inlet velocity of 15 m/s.

### Fix 2: Add Bounding for `k` and `epsilon` Equations

Add bounding constraints to prevent extreme values during the initial transient:

In **`0/k`**, add after the `internalField` line:
```
bounds
{
    min 0;
    max 0.06;
}
```

In **`0/epsilon`**, add after the `internalField` line:
```
bounds
{
    min 0;
    max 14855;
}
```

### Fix 3: Reduce Relaxation Factors in `fvSolution`

The current `fvSolution` uses very aggressive relaxation (default Euler scheme without explicit relaxation factors). For PIMPLE with the default Euler scheme, add under-relaxation to stabilize the initial transient:

In **`system/fvSolution`**, wrap the solvers for `U`, `k`, `epsilon`, and `nuTilda` with relaxation factors, e.g.:

