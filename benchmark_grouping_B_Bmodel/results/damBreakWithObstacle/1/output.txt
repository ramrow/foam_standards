In OpenFOAM 10 (Foundation), `subsetMesh` works differently than newer versions. The command syntax is:

```
subsetMesh <cellSet> -patch <existingPatch>
```

It does **NOT** accept a `-patch newname` flag to create a brand new patch. Instead, it requires an **existing** patch name to place the newly exposed internal faces onto. And critically, the cellSet argument comes **first**, not last.

So the correct invocation should be:

```

```

But wait — looking more carefully at the reference case structure and typical usage, in OpenFOAM 10 (Foundation), `subsetMesh` actually takes the form:

```
subsetMesh -overwrite <cellSet> -patch <existingPatch>
```

The crucial difference is that `<existingPatch>` must already exist in the mesh. The `-patch` flag in OpenFOAM 10 assigns the exposed faces to an **already-present** patch rather than creating a new one.
