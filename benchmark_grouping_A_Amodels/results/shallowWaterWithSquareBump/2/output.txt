## Error Diagnosis

The error is:

```
Unknown symmetric matrix preconditioner DILU

Valid symmetric matrix preconditioners :
5
(
DIC
FDIC
GAMG
diagonal
none
)
```

This error occurs during the solve of the `h` field (water depth). The `h` field produces a **symmetric matrix** (because the Laplacian operator on a scalar field yields a symmetric matrix). The preconditioner `DILU` (Diagonal Incomplete LU decomposition) is only valid for **asymmetric** matrices. For symmetric matrices, the equivalent and valid preconditioner is `DIC` (Diagonal Incomplete Cholesky).

Looking at the `fvSolution` file:

```
solvers
{
    h
    {
        solver          PBiCGStab;
        preconditioner  DILU;
        ...
    }
}
```

The `h` solver uses `DILU`, which is inappropriate for the symmetric matrix it generates. Additionally, the `hFinal` solver inherits from `h` via `$h;` and thus suffers from the same issue.

## Solution

**Modify `system/fvSolution`** to change the preconditioner for the `h` (and `hFinal`) solvers from `DILU` to `DIC`. The corrected solvers sub-dictionary should be:

```
solvers
{
    h
    {
        solver          PBiCGStab;
        preconditioner  DIC;
        tolerance       1e-12;
        relTol          0.01;
    }

    hFinal
    {
        $h;
        relTol          0;
    }

    hU
    {
        solver          PBiCGStab;
        preconditioner  DILU;
        tolerance       1e-12;
        relTol          0.01;
    }

    hUFinal
    {
        $hU;
        relTol          0;
    }
}
```

Key changes:
1. **Change `preconditioner DILU` to `preconditioner DIC`** for the `h` solver — `DIC` is the symmetric counterpart of `DILU` and is the standard choice for the pressure-like equation solved for `h`.
2. **Leave `hU` unchanged** — the momentum equation produces an asymmetric matrix, so `DILU` remains valid for `hU`.

This follows the pattern used in the squareBump tutorial, where the `h` solver uses `DIC` (symmetric) and the `hU` solver uses `DILU` (asymmetric). The `hFinal` solver inherits from `h` via `$h;`, so it will automatically pick up the `DIC` preconditioner as well.
