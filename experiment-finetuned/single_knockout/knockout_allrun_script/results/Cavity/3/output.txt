The fact that the error persists across multiple attempts suggests that the issue is not with the `controlDict` format itself, but rather with how the solver is being invoked.

## Root Cause

Looking more carefully at the error message and the `controlDict`, the issue is that the `pisoFoam` solver being used is likely a wrapper script or an older version that automatically adds these deprecated command-line flags (`-writep`, `-writePhi`, `-withFunctionObjects`) based on the contents of `controlDict`. In OpenFOAM 7+, these flags were removed, but some wrapper scripts or build systems still try to pass them.

The key insight is that the error shows these flags are being passed **automatically** by the invocation mechanism, not manually by the user. This means the problem is likely with the build system or wrapper script that calls `pisoFoam`.

## Proposed Solutions

### 1. Check for and remove any wrapper scripts or build system configurations

The user should look for:
- A `Makefile` or `wmake` configuration that adds these flags
- A shell script or batch file that calls `pisoFoam` with additional flags
- Any custom build system that wraps the `pisoFoam` invocation

In particular, check if there's a `wmake` rule or similar that adds `-writep`, `-writePhi`, or `-withFunctionObjects` based on the `controlDict` contents. These flags should be removed from any such wrapper scripts.

### 2. Use the direct `pisoFoam` invocation without any wrapper

The user should run the simulation steps directly from the command line without any intermediate scripts:

```

```

If a wrapper script exists (e.g., `./allrun`), they should examine it and remove any lines that add `-writep`, `-writePhi`, or `-withFunctionObjects` to the `pisoFoam` command.

### 3. Check the `controlDict` for any remaining OpenFOAM 2.x syntax that might trigger these flags

Although the `writeControl` entries look correct, there might be other entries in `controlDict` that use the old OpenFOAM 2.x syntax and cause the wrapper to add these flags. Specifically, check for:

- Any `writeControl` entries that use syntax like `writeControl timeStep;` with additional parameters that might trigger flag generation
- Any `writeInterval` or related entries that use old-style syntax

The cleanest approach is to simplify the `controlDict` write control section to the bare minimum required for OpenFOAM 7+:

```
writeControl    timeStep;
writeInterval   100;
purgeWrite      0;
```

And ensure no other entries in `controlDict` reference write-related functionality that might trigger the deprecated flags.

### 4. Alternative approach: Use `writeControl` with `writeInterval` and ensure the `pisoFoam` invocation is clean

The most direct solution is to ensure that when `pisoFoam` is invoked, it is called **without any additional command-line flags**. The error message shows these flags are being added automatically, which means the build system or wrapper is responsible. The user should:

1. Locate the exact command being used to invoke `pisoFoam`
2. Remove any `-writep`, `-writePhi`, or `-withFunctionObjects` flags from that command
3. Ensure the `controlDict` only contains the modern OpenFOAM 7+ write control syntax shown above

## Summary
