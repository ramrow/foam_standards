
Alternatively, add a `pRefCell` entry directly into the `p_rgh` boundary field dictionary (in `0/p_rgh`), adding something like:

```

pRefValue       0;
```

to the root level of the `p_rgh` dictionary (not inside any single patch entry). But the cleaner approach is Option A in `fvSolution`.

The error message shows that `subsetMesh` is receiving a patch name as a positional argument (rather than `-patch`) and is interpreting it as the cellSet name, which fails validation.

The most likely scenario is that the `submit.sh` script or some automation tool is calling `subsetMesh` with incorrect arguments.
