## Error 1: `log.blockMesh` — Block is "inside-out"

**Error Message:**
```
Block hex (0 1 2 3 4 5 6 7) (100 60 1) simpleGrading (1(1) 1(1) 1(1)) is inside-out
```

**Diagnosis:**

The vertex ordering in the hex block definition creates a negative volume, meaning the vertices don't follow the right-hand rule convention required for OpenFOAM hex blocks.

Looking at the current vertex definitions:
- **Front face (z = 0):** 0=(-1,-0.4,0), 1=(-1,0.4,0), 2=(1,0.4,0), 3=(1,-0.4,0)
- **Back face (z = 0.1):** 4=(-1,-0.4,0.1), 5=(-1,0.4,0.1), 6=(1,0.4,0.1), 7=(1,-0.4,0.1)

For a hex block `(0 1 2 3 4 5 6 7)`, the first four vertices form one face and the second four form the opposite face. The issue is that the winding order needs to be checked:

- Face 0→1→2→3 (front): goes from left → up → right → down ✓ (consistent CCW when viewed from outside)
- But face 4→5→6→7 (back): also goes left → up → right → down

In OpenFOAM's hex convention, the back face should have the **opposite** winding direction relative to the front face so that the block has outward normals pointing away from the interior. The back face vertices need to be reordered.

**Proposed Fix:**

Change the back face vertex ordering from `(4 5 6 7)` to `(7 6 5 4)`. This reverses the back face winding, making the block properly oriented with outward-pointing normals.

So the block definition should be:
```
hex (0 1 2 3 7 6 5 4) (100 60 1) simpleGrading (1 1 1)
```

This ensures the front face (0,1,2,3) and back face (7,6,5,4) have opposite windings, satisfying OpenFOAM's hex block convention.
