# Error Diagnosis and Resolution

## Error Analysis

The error message is clear:

```
keyword viscosityModel is undefined in dictionary "constant/physicalProperties"
```

This means the `pisoFoam` solver expects a `viscosityModel` entry in the `physicalProperties` file, but the current file only contains a simple `nu` (kinematic viscosity) entry.

Looking at the reference cavity case and the standard OpenFOAM `pisoFoam` solver behavior, when using a `transportProperties` approach (which is what the reference case uses), the `physicalProperties` file should define a `viscosityModel` — typically `constant` — and the `nu` value is read from there automatically.

However, the user requirement states **"The viscosity (`nu`) is set as constant with a value of 1e-05 m²/s."** So we need to keep `nu = 1e-05`.

## Recommended Fixes

### Option 1 (Preferred): Add `viscosityModel` to `physicalProperties`

Modify `constant/physicalProperties` to include a `viscosityModel`:

```
FoamFile
{
    version     2.0;
    format      ascii;
    class       dictionary;
    location    "constant";
    object      physicalProperties;
}

viscosityModel  constant;

nu              [0 2 -1 0 0 0 0] 1e-05;
```

This tells the solver to use a constant viscosity model, while still setting `nu` to the required 1e-05 value.

### Option 2: Remove `physicalProperties` and put `nu` directly into `transportProperties`

If you prefer to follow the typical OpenFOAM convention where `transportProperties` handles the viscosity model, create a `transportProperties` file in the `constant/` directory:

```
FoamFile
{
    version     2.0;
    format      ascii;
    class       dictionary;
    location    "constant";
    object      transportProperties;
}

nu              [0 2 -1 0 0 0 0] 1e-05;
```

Then **remove** the `physicalProperties` file entirely. The `pisoFoam` solver reads `nu` from `transportProperties` and constructs the effective viscosity internally. However, since the user explicitly specified `physicalProperties`, Option 1 is more consistent with their setup.

### Note on Turbulence Fields

Since the user requires **"use k epsilon turbulence model"**, keeping the `k`, `epsilon`, `nut`, and `omega` field files in `0/` is appropriate — these will remain unused by `pisoFoam` (laminar solver ignores them) but won't cause harm. Just ensure they don't conflict with future RANS simulations.

## Summary

The single fix needed is adding `viscosityModel constant;` to the `constant/physicalProperties` file. This satisfies the solver's expectation while preserving the user-specified `nu = 1e-05`.
