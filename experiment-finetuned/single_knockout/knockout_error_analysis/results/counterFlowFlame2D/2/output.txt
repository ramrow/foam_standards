The compiler cannot find a mixture type named `reactingMixture` in the current OpenFOAM installation, so the solver aborts before the reacting flow can be initialised.

`reactingMixture` is defined in the *reacting* library (`reactingMixture` lives in the `reactiveMixture`/`reactingMixture` module). When only the `hePsiThermo` thermophysical model is used, that library is not automatically loaded, so the dynamic‑code filter cannot resolve the symbol.

}
```

- `reactingMixture` is the library that provides the `reactingMixture` type used by `hePsiThermo`.  
- Declaring it in `controlDict` forces the dynamic‑code loader to load the library early, making the type visible when the template is compiled.  
- No other entries (e.g., `thermoType`, `chemistryProperties`, boundary conditions) need to be edited; the solver will then be able to resolve `reactingMixture` and continue with the simulation exactly as defined in the reference case.
