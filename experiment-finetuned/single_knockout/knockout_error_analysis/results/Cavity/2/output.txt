When the **`kEpsilon`** turbulence model is used, the wall‑function for **`epsilon`** must be **`epsilonWallFunction`**, which internally expects the **`nut`** field to be of type **`nutWallFunction`**.  
Because **`nut`** is still declared as **`calculated`** (or is being cast to `nutWallFunction` while its internal type is still `calculated`), the cast fails and the solver aborts.

The root cause is therefore a **type‑mismatch between the turbulence model and the wall‑function patches**:

| Patch | Current type | Required type for `kEpsilon` |
|-------|--------------|------------------------------|
| `epsilon` | `epsilonWallFunction` | `epsilonWallFunction` (needs `nut` of type `nutWallFunction`) |
| `nut`     | `calculated` (or undefined) | `nutWallFunction` |

---

   nut
