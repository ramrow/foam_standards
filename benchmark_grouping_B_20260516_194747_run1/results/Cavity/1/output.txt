## Root Cause

In newer versions of OpenFOAM (Foundation version), the `physicalProperties` file format has changed. Instead of simply specifying `nu`, the solver requires a `viscosityModel` sub-dictionary (or entry) to determine how viscosity is computed. For a constant kinematic viscosity, the solver needs to know the `viscosityModel` (e.g., `constant`) and the corresponding `nu` value.

Looking at the current `physicalProperties`:

```

```

This direct `nu` specification is the old format and is not recognized by `pisoFoam` in this OpenFOAM version.

## Proposed Solution

**Modify `constant/physicalProperties`** to use the new format compatible with `pisoFoam`. Specifically, add a `viscosityModel` entry set to `constant` and include the `nu` value within a `nu"` sub-dictionary (or use the flat `constant nu;` syntax depending on the OpenFOAM version).

The recommended format should look like:

```
FoamFile

    format      ascii;
    class       dictionary;
```

If only the `nu` keyword alone doesn't work, try the full specification with `viscosityModel constant;` explicitly set. The key point is that `pisoFoam` looks for `viscosityModel` in `physicalProperties` — without it, the solver cannot proceed.
