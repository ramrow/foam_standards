- Time = 0.01s: max Courant = 2.066

### Root Cause

The mesh has very small cells near the step (especially in the refined region), and with `deltaT = 0.002`, the CFL condition is violated. ### Solution

**In `system/controlDict`**, change `adjustTimeStep` from `no` to `yes`. This allows the solver to automatically reduce `deltaT` to maintain stability based on the Courant number. Keep `maxCo` at a safe value (e.g., 0.5 or 0.7) to ensure stability.

The key change:
```

maxCo           0.5;
maxDeltaT       0.001;
```

This will allow the solver to dynamically reduce the timestep when the Courant number approaches the stability limit, preventing the SIGFPE crash in the `sqrt` operation.

## Error: Floating Point Exception (SIGFPE) in `sqrt` function

### Diagnosis

The crash is occurring in `Foam::sqrt` at around Time = 0.012s, with extremely high Courant numbers (max ~2.07). This is a numerical instability issue in the explicit compressible solver. The `adjustTimeStep yes` with various `maxCo` and `maxDeltaT` settings should prevent this, but the crash is still happening.

Looking at the current state, the simulation is running with `adjustTimeStep yes`, but the crash is occurring very quickly. This suggests that the adaptive timestep is not reducing `deltaT` fast enough near the sharp step. The Courant number is growing beyond stability limits almost immediately.

### Root Cause

The mesh has a very sharp step at x=0.6. The cell adjacent to the step face has a very small height (0.2 in y, with 8 cells in y, so ~0.025 per cell). With a velocity of 3 m/s and a small cell size, the local Courant number can easily exceed 1 even with conservative settings. The adaptive timestep is not reacting quickly enough to the sudden increase in Courant number near the step.

### Solution

**In `system/controlDict`**, add a `maxDeltaT` that is significantly smaller to provide more headroom for stability:

```
maxDeltaT       0.000005;
```

**In `system/controlDict`**, also add `maxCoAdjustmentFactor 0.01;` to make the adaptive reduction more aggressive when the Courant number approaches the limit:

```
maxCoAdjustmentFactor 0.01;
```

**In `system/controlDict`**, consider reducing `maxCo` to a more conservative value like `0.1`:

```
maxCo           0.1;
```

The combination of these changes — especially the extremely small `maxDeltaT 0.000005` and very aggressive `maxCoAdjustmentFactor 0.01` — should allow the adaptive timestep to reduce `deltaT` rapidly enough near the sharp step to maintain numerical stability, preventing the SIGFPE crash in the `sqrt` operation. The `maxCo 0.1` provides an additional safety margin.

This approach is different from previous attempts by using an even smaller `maxDeltaT` (0.000005) combined with a very aggressive `maxCoAdjustmentFactor` (0.01) and conservative `maxCo` (0.1) to ensure the timestep reduces fast enough near the step instability.
