The valid asymmetric preconditioners are `DILU`, `GAMG`, `diagonal`, `none`. So `DIC` is rejected here even though it's an asymmetric solver (`PBiCGStab`).

## Root Cause

The `hU` field (momentum/hU) produces an asymmetric matrix. The preconditioner `DIC` is not recognized as valid for asymmetric systems in this OpenFOAM version. The solver explicitly wants `DILU` (not `DIC`) for asymmetric systems.

## Solution

In `system/fvSolution`, change the preconditioner for the `hU` solver from `DIC` to `DILU`:

Current:
```
hU
{
    solver          PBiCGStab;
    preconditioner  DIC;
    tolerance       1e-12;
    relTol          0.01;
}
```

Should become:
```
hU
{
    solver          PBiCGStab;
    preconditioner  DILU;
    tolerance       1e-12;
    relTol          0.01;
}
```

Keep the `h` solver unchanged (it already uses `DIC` which is correct for the symmetric system produced by the scalar `h` field). The `hFinal` entry inherits from `$h` so it will also use `DIC` automatically, which is correct.

This is the opposite fix from Attempt 4 — previously `DIC` was applied to the symmetric `h` equation (correct), and now `DILU` is needed for the asymmetric `hU` equation.
