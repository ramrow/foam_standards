## Root Cause: Missing Pressure Reference

In OpenFOAM's `interFoam` solver (which solves compressible Navier-Stokes equations), the pressure equation requires a reference point or cell to avoid a singular matrix. The error message is very clear: **"Please supply either pRefCell or pRefPoint"**.

Since the mesh has no explicit `pRefCell`/`pRefPoint` specified anywhere, and the `p_rgh` boundary conditions already include `fixedFluxPressure` entries (which should work), the issue is that `interFoam` internally looks up the pressure field differently than `simpleFoam`.

---

## Proposed Solutions

### Option A: Add `pRefCell` and `pRefValue` to `fvSolution` (Recommended)

Add a `pRefCell` and `pRefValue` entry inside the `PIMPLE` sub-dictionary in `system/fvSolution`. For example:

```
PIMPLE
{
    momentumPredictor   no;
    nOuterCorrectors    1;
    nCorrectors         3;
    nNonOrthogonalCorrectors 0;

    pRefCell            0;
    pRefValue           0;
}
```

Alternatively, add them outside PIMPLE but inside the main dictionary:

```
pRefCell        0;
pRefValue       0;
```

Place `pRefCell` and `pRefValue` at the same level as `solvers`, `relaxationFactors`, etc.

### Option B: Modify `0/p_rgh` to include `pRefCell`/`pRefValue` entries

If the solver reads the `p_rgh` field directly for the reference, ensure the `0/p_rgh` file includes a `pRefCell`/`pRefValue` entry in its `boundaryField`. For example, add to the root level of the `p_rgh` file:

These specify that cell index 0 of the `p` (pressure) field serves as the reference cell with value 0. This is the standard way to handle this for `interFoam` in OpenFOAM 10.
