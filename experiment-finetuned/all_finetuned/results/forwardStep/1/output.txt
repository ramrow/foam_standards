## Error Analysis

```

```

This means `sqrt()` is being called on a negative number — likely computing the speed of sound `sqrt(gamma * p / rho)`, where pressure or density has gone negative due to numerical instability.

## Root Cause

The simulation is **too unstable**. With `deltaT = 0.002` and the mesh resolution, the CFL number is too high for stability. The Courant numbers shown are ~0.85-0.89, which is approaching the stability limit for this explicit solver. Additionally, the mesh has relatively coarse resolution (20×20 upstream, 80×20 downstream) which doesn't adequately resolve the shock near the step.

## Proposed Solutions

### 1. Reduce the time step significantly

Change `deltaT` from `0.002` to something much smaller like `0.0005` or even `0.00025` in `controlDict`. This will reduce the CFL number and improve stability.

### 2. Add sub-cycling or adjust the solver's CFL control

In `controlDict`, enable adaptive time stepping to keep the CFL below 0.5:

```
adjustTimeStep  yes;
maxCo           0.5;
```

This allows the solver to automatically reduce the time step when needed.

### 3. Improve the initial condition for pressure

The initial pressure of `1 Pa` combined with the velocity of `3 m/s` creates a very low Mach number flow (~0.005), but the combination can still cause issues. Ensure the pressure field is consistent with the initial velocity field.

### 4. Use a more dissipative reconstruction scheme

In `fvSchemes`, change the divergence schemes to be more diffusive:

```

{

### 5. Increase the number of relaxation sweeps

relTol          0.1;
```
