## Error: `log.shallowWaterFoam` — Unknown symmetric matrix preconditioner DILU

**Error:**
```
Unknown symmetric matrix preconditioner DILU

Valid symmetric matrix preconditioners :
5
(

FDIC

diagonal
none
)
```

**Diagnosis:** The `h` field equation produces a **symmetric matrix**. The preconditioner `DILU` (Diagonal Incomplete LU decomposition) is only valid for **asymmetric** matrices. For symmetric matrices, the equivalent is `DIC` (Diagonal Incomplete Cholesky).

Looking at the `fvSolution` file, both `h` and `hU` solvers use:
```
preconditioner  DILU;
```

For the `h` equation (a scalar transport-like equation), the matrix is symmetric, so `DILU` → `DIC` is the fix. For `hU`, the momentum equation typically produces an asymmetric matrix, so `DILU` remains appropriate there.

**Resolution:** In `system/fvSolution`, change the preconditioner for the `h` solver (and consequently `hFinal` which inherits from `$h`) from `DILU` to `DIC`:

Change:
```
h
{
    solver          PBiCGStab;
    preconditioner  DILU;
    tolerance       1e-12;
    relTol          0.01;
}
```

to:
```
h
{
    solver          PBiCGStab;
    preconditioner  DIC;
    tolerance       1e-12;
    relTol          0.01;
}

hFinal
{
    $h;
    relTol          0;
}
```

The `hU` solver can keep `DILU` since the momentum equation typically yields an asymmetric matrix.
